## Небольшой конспект

1. ООП - объектно ориентированное программирование - методология программирования, основанная на представлении программы в виде совокупности объектов, каждый из которых является экземпляром определённого класса, а классы образуют иерархию наследования. (Основные понятия в ооп - это инкапсуляция, наследование и полиморфизм)
2. Инкапсуляция - объединение данных и методов, работающих с ними, в классе + разграничении доступа различных частей программы к внутренним компонентам друг друга 
3. Полиморфизм – это способность программы идентично использовать объекты с одинаковым интерфейсом без информации о конкретном типе этого объекта.
4. Наследование - базовый принцип ООП, позволяет одному классу (наследнику) унаследовать функционал другого (родительского) класса. Другими словами - генерализация/обобщение
5. Реализация - определение интерфейса (определение всех его функций)
6. Наследоваться можно только от одного класса (extends), реализовывать можно много интерфейсов (implements)
7. Ассоциация - это отношение, при котором объекты одного типа неким образом связаны с объектами другого типа. Агрегация и композиция являются частными случаями ассоциации.
8. Композиция -  отношение HAS A, то есть отношение "имеет". При удаление объекта который связан с другим объектом композиции (удаляя объест который содержит другой объект), второй объект также удалится
9. Агрегация -  отношение HAS A, то есть отношение "имеет". При удаление объекта который связан с другим объектом агрегацией (удаляя объест который содержит другой объект), второй объект не удалится
